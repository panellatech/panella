# K1 Slot Registry Taxonomy

This is a domain-level breadth checklist for the K1 resolver registry. It is
intentionally generic personal-memory coverage and is not derived from any
evaluation or goldset content.

| Domain | Minimum slots |
| --- | ---: |
| identity | 4 |
| health | 5 |
| household | 4 |
| work | 5 |
| transport | 4 |
| devices | 4 |
| digital-services | 4 |
| schedule | 4 |
| social | 4 |
| pets | 3 |
| hobbies | 3 |
| finance-adjacent | 4 |
| safety-constraints | 4 |
| **Total** | **52** |

The registry must meet or exceed each minimum with canonical, generic slot
domains. High-risk slots carry a lexicon and narrowly scoped deny-neighbor
notes; those notes are routing/prompt boundaries, not permission to write.
